# 開發歷程：product-image-resizer

> 給電商店家的批次商品圖轉檔工具網站——零上傳、零營運成本

---

## 🎯 需求起源（2026-07-07）

電商平台的店家常有一批照片要上架，但系統對商品圖有尺寸與檔案大小要求。店家自己不會（或懶得）用修圖軟體逐張處理，需要一個工具網站：

1. **批量**處理照片
2. **等比例縮放**，不能破壞比例（不變形）
3. 符合系統的**尺寸**與**檔案大小**上限
4. 用**免費流量**跑，而且要能「限制流量」避免太多人用導致站主要付費

## 💡 關鍵決策：純前端處理（Client-side）

**問題**：如果照片上傳到伺服器處理，流量與運算成本跟用量成正比，免費方案很快爆掉，還要做限流。

**評估過的方案**：
| 方案 | 流量成本 | 爆量風險 |
|---|---|---|
| 後端處理（Cloud Run / Lambda） | 上傳+下載+運算全算你的 | 高，需限流與帳單警報 |
| ✅ **瀏覽器內處理（Canvas API）** | **趨近於零**（只載一次靜態頁） | **無**，運算用店家自己的電腦 |

**決定**：純前端。照片從頭到尾不離開使用者的瀏覽器——「限制流量」的需求直接消失，因為根本沒有會產生費用的流量。附帶好處：不用等上傳、隱私安全（商品圖不經第三方）。

**託管選擇**：GitHub Pages（免費 100GB/月頻寬、**超量只限流不收費、沒有帳單機制**）。Cloudflare Pages 也可（頻寬無上限），但用戶已有 GitHub 工作流程，Pages 部署 = git push，最順。

**「限制對象」取代「限制流量」**：如果之後不想讓工具外流，加一組前端通行碼即可（跟店家公告，改版換碼）。首版先不加。

## 🛠️ 實作要點

- **等比縮放**：`scale = min(targetW/w, targetH/h, 1)`——只縮小不放大，比例永遠不變
- **兩種輸出模式**：
  - `pad`（預設）：等比縮放後置中、補底色（預設白），輸出固定尺寸（如 1000×1000）——商品圖最常見的需求
  - `fit`：等比縮至長邊，輸出保持原比例
- **畫質**：大圖一次縮到小尺寸會糊 → **分段減半縮小**（step-down halving）+ `imageSmoothingQuality: "high"`
- **手機直拍轉向**：`createImageBitmap(file, { imageOrientation: "from-image" })` 自動處理 EXIF，失敗退回 `<img>` 載入
- **壓到大小上限**：`canvas.toBlob` 品質從 0.92 逐級降到 0.4，取第一個 ≤ 上限的結果；PNG 沒有品質參數，超標直接提示改用 JPG
- **批次下載**：JSZip 打包，同名檔自動加流水號
- **規格未定的處理**：用戶尚未提供平台的正式圖片規格 → 尺寸/大小上限/格式全部做成頁面可調設定（預設 1000×1000、白底補邊、≤1MB、JPG），規格確定後再鎖成預設值
- `noindex`：不讓搜尋引擎收錄，工具只給自家店家用

## 📋 規格預設值

| 項目 | 預設 | 可選 |
|---|---|---|
| 尺寸 | 1000×1000 | 800 / 1200 / 自訂 |
| 縮放 | 等比＋白底補邊 | 等比縮至長邊 |
| 大小上限 | 1 MB | 300KB / 500KB / 2MB / 不限 |
| 格式 | JPG | WebP / PNG |

## 📣 定位調整（2026-07-07）：從「內部工具」變「行銷推廣」

評估過前端通行碼方案（SHA-256 雜湊 + localStorage 記憶）後，用戶決定**不加鎖、完全開放**——因為純前端架構外人用了也零成本，開放反而是 OrderPally 的品牌曝光。配合調整：
- 移除 `noindex`，改加 SEO meta description，`<title>` 加關鍵字「商品圖批次縮圖壓縮工具」讓搜尋引擎收錄

## 📐 正式規格定案（2026-07-07）

用戶問到 OrderPally 官方規格：**尺寸不限、檔案 ≤ 2MB**。調整：
- 預設改「維持原尺寸（只壓縮檔案）」+ 上限 2MB；原尺寸模式下隱藏縮放方式/底色選項（無意義）
- 尺寸選項仍保留（800/1000/1200/自訂），店家想順便縮圖也行
- **保險機制**：極大圖（例如單眼原檔）壓到品質下限 0.4 仍超過 2MB 時，自動每輪縮小 20% 尺寸再壓，直到符合（最多 6 輪或短邊 300px 止）——規格尺寸不限所以縮尺寸可接受
- 原尺寸模式輸出檔名後綴 `_compressed`
